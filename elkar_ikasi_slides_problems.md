# The KTool Problem

## Where we are now

### Data is disconnected
- Agents, projects, perceptions, channels, information sit in separate tables
- No way to see the system as a whole
- "Who is connected to whom?" requires manual cross-referencing

### Analysis is manual
- Each question requires a new SQL query or spreadsheet
- Structural metrics (centrality, fragility, k-core) don't exist
- Narrative patterns buried in raw quote text — unreadable at scale

### Insight is invisible
- No network map
- No "what if" simulation
- No financial × perception cross-reference
- No structural change assessment

### Governance questions go unanswered
- "Where would a small intervention have the biggest effect?" → unknown
- "What's blocking change from propagating?" → invisible
- "Is the ecosystem locked into its current configuration?" → untested
- "Which narratives are funded vs which are excluded?" → unreported

## Bottom line

**KTool stores data. It does not model the system.**

The pipeline adds the missing layer: from relational data → network model → structural diagnosis → actionable hypothesis.
